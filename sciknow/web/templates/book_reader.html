          ○ Idle
          
        
          
        
          
          The book plan is a 200–500 word document defining the central thesis,
          scope, intended audience, and key terms. It is injected into every
          book write / autowrite call so all chapters stay
          aligned with the same argument. Edits autosave after 1.2 s of no typing.
        
          
        
          
          
          
        
          
        
          
          
          
        
          
        
          
          
          
        
          
        
          
          
          
          
            
            
            
            
            
          
          
          
          
            Each section gets a proportional share: a 4-section chapter at 6000
            words asks the writer for ~1500 words per section. In autowrite,
            length becomes a 7th scoring dimension — drafts under ~70% of
            target trigger a targeted expansion revision.
          
          
        
          
      
          
      
      
          
        
          
          Outline proposes a chapter structure for the book from your paper
          corpus. The LLM generates 3 candidate outlines (temperature-diversified),
          scores each for breadth + section-count variance, picks the winner, and
          then density-resizes each chapter’s section list by counting corpus
          evidence per topic. Insert is additive: existing chapters
          and drafts are never touched — new chapters are appended with fresh
          numbers. Run again to re-roll when the first pass isn’t right.
        
          
        
          
          Outline vs Review:
          Outline plans the book’s chapter/section structure from the corpus (no drafts needed).
          Review (in the draft toolbar) critiques the prose in a single existing draft
          across 5 dimensions (groundedness, completeness, accuracy, coherence, redundancy).
          Different inputs, different outputs, different stages of the workflow.

           AI Actions — what each button does
          
      
    
          
    
          
      
          
        sciknow’s book workflow is a pipeline: plan the shape of the book,
        then write drafts, then critique + fix.
        Each button below belongs to one of those stages. Every action maps 1:1 to a
        sciknow book ... CLI command, so what you see in the GUI is exactly what
        the terminal does.
      
          

      
          📝 Planning — shape the book before writing
          
      
          
        
          Outline
          Plan modal → Outline tab          		
          
            Proposes a chapter structure from your paper corpus. Generates 3 candidate outlines
            at rising temperatures, scores each for breadth + section-count variance, picks the
            winner, then density-resizes each chapter’s section list by counting corpus
            evidence. Additive: never touches existing chapters or drafts.
            Use when: bootstrapping a new book or expanding the plan.
            Produces: new book_chapters rows. CLI: book outline.
          
          
        
          Leitmotiv (Book plan)
          Plan modal → Book tab          		
          
            The 200–500 word thesis document that gets injected into every write /
            autowrite call so all chapters stay aligned. Edit it by hand, or click
            Regenerate with LLM.
            Use when: the book’s central argument drifts or needs sharpening.
            CLI: book plan.
          
          
      
          

      
          ✎ Writing — produce draft prose
          
      
          
        
          AI Write
          
            Single-pass draft of the current section. Retrieves sources, writes the prose, stops.
            Fast, but doesn’t self-improve. Use when: you want a fresh baseline you’ll
            hand-edit. CLI: book write.
          
          
        
          AI Autowrite
          
            Full convergence loop: write → score → verify claims → revise → rescore
            for up to 3 iterations or until overall score hits 0.85. Uses the writer model for
            prose and AUTOWRITE_SCORER_MODEL (gemopus4) for discrimination.
            Use when: you want a polished draft, not a starter.
            CLI: book autowrite.
          
          
        
          Edit
          
            Manual in-browser markdown editor with autosave, KaTeX math, and inline figure
            thumbnails. Use when: humans need to be humans.
          
          
      
          

      
          🔍 Critique — find what’s wrong with a draft
          
      
          
        
          AI Review
          
            Single-pass critic across 5 dimensions (groundedness, completeness, accuracy,
            coherence, redundancy). Produces structured feedback with quotes + actionable
            suggestions. Writes to drafts.review_feedback.
            Use when: you have a draft and want to know where to fix it.
            Different from Outline: Outline builds structure; Review critiques prose.
            Uses BOOK_REVIEW_MODEL (see Book Settings → Models for the current pick). CLI: book review.
          
          
        
          Adversarial review
          
            Harsher critic — forces ≥ 10 concrete issues, never graded.
            Doesn’t overwrite the normal review_feedback.
            Use when: the standard review feels too gentle.
            CLI: book adversarial-review.
          
          
        
          Edge cases
          
            Exhaustive boundary-condition hunter: walks every scope boundary, counter-case,
            causal alternative, and quantitative limit. Structured JSON output.
            Use when: you need the draft to survive hostile readers.
            CLI: book edge-cases.
          
          
        
          Ensemble Review
          
            N independent NeurIPS-rubric reviewers (default 3, T=0.75, rotating neutral/pessimistic/
            optimistic stance) + a meta-reviewer that medians scores and unions findings.
            Costs ~N× the single-review. Use when: a single reviewer felt flaky.
            CLI: book ensemble-review.
          
          
        
          Argue (map claim)
          
            For any claim you type, builds a SUPPORTS / CONTRADICTS / NEUTRAL evidence map
            from your corpus. Use when: you want to see how solid a single
            assertion is. CLI: book argue.
          
          
        
          Gaps
          
            Identifies topics the book’s plan names but the drafts don’t cover yet.
            Use when: a chapter feels thin and you’re not sure why.
            CLI: book gaps.
          
          
      
          

      
          ✓ Verification — check citations actually support claims
          
      
          
        
          Verify
          
            Sentence-level LLM verifier: classifies each [N] claim as SUPPORTED /
            EXTRAPOLATED / MISREPRESENTED / OVERSTATED and returns a groundedness + hedging
            fidelity score. CLI: book verify-citations.
          
          
        
          Verify Draft
          claim-atomization          		
          
            Splits each sentence into atomic sub-claims (regex first, LLM fallback for compound
            sentences), NLI-scores each sub-claim against source chunks, and surfaces
            mixed-truth sentences — where part of the sentence is supported
            and part isn’t. Catches what the sentence-level verifier misses.
            Read-only. CLI: book verify-draft.
          
          
        
          Align Citations
          
            Post-pass that (conservatively) remaps [N] markers to the chunk that
            actually entails the sentence, when the currently-cited chunk has
            entailment < 0.5 AND the top chunk beats it by ≥ 0.15.
            Use when: verify reports mismatched citations.
            CLI: book align-citations.
          
          
        
          Insert Citations
          
            Two-pass LLM adds [N] markers where the prose asserts something
            source-worthy but has no citation. Saves a new draft version.
            Use when: hand-written prose is missing its citation layer.
            CLI: book insert-citations.
          
          
      
          

      
          ⚠️ Fixing — apply feedback to the draft
          
      
          
        
          AI Revise
          
            Reads the latest review_feedback saved on the draft and rewrites the
            prose to address it. Requires: a Review (or Autowrite, which reviews
            internally) has been run first. Use when: you’re happy with the
            critique and want the draft to act on it. CLI: book revise.
          
          
      
          

      
          📊 Diagnostics — inspect the autowrite trajectory
          
      
          
        
          Scores
          
            Shows the 5-dimension score trajectory across autowrite iterations — which
            scores rose, which plateaued, which triggered revisions. Read-only.
          
          
        
          Bundles
          
            Chapter- or book-wide snapshots. Safety net for autowrite-all-sections
            runs — snapshot before, restore if you hate the result.
            CLI: book snapshot / snapshot-restore.
          
          
        
          Chapter reader
          
            Read-only continuous-scroll view of the whole chapter so you can feel the flow
            without the editor chrome.
          
          
      
          

      
          🛡️ Typical workflow
          
      
            
        
          1. Outline (Plan modal) → book has chapters
            2. 
        
          2. AI Autowrite per section → first drafts exist, already reviewed + revised once
            3. 
        
          3. Verify Draft + Align Citations → confirm citations hold up
            4. 
        
          4. AI Review (or Adversarial / Ensemble) → structured critique
            5. 
        
          5. AI Revise → apply feedback
            6. 
        
          6. Repeat 4–5 until happy, then Bundle snapshot the chapter
            7.
